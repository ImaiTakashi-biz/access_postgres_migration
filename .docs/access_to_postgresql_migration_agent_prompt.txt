# 依頼内容：AccessデータベースをPostgreSQLへ忠実に移行する

## 目的

Accessデータベース（.accdb）をPostgreSQLへ移行してください。

複数のAccessシステムを順次移行していくため、`.docs` フォルダ配下の対象フォルダは、私が都度指定します。
指定された対象フォルダの内容を確認し、Accessデータベースの構造・テーブル・カラム・データ型・データ内容をできる限り忠実にPostgreSQLへ移行してください。

## 重要な制約

以下は必ず守ってください。

* Access側のテーブルを勝手に削除しない
* Access側のテーブルを勝手に統合しない
* カラムを勝手に削除しない
* カラム名の意味を勝手に変えない
* データを勝手に加工・集約しない
* 不明な項目は推測で変更せず、コメントや資料に「要確認」として残す
* 既存コードを破壊的に変更しない
* まずは現行Access DBの忠実なPostgreSQL移行を最優先とする

## 対象フォルダ

今回の対象フォルダは以下です。

```text
【ここに対象フォルダパスを貼り付け】
例：.docs/外観検査記録DB
```

このフォルダ内にある資料・解析結果・Accessファイル・設定ファイルを確認してください。

## PostgreSQL接続情報

PostgreSQLの接続URLは、対象フォルダ内の `.env` に記載されています。

例：

```text
DATABASE_URL=postgresql://user:password@localhost:5432/database_name
```

`.env` を読み取り、そこに記載されたPostgreSQLへ移行してください。

## 実施してほしいこと

### 1. 対象フォルダの内容確認

対象フォルダ内の以下を確認してください。

* Accessデータベースファイル
* 解析済みMarkdown資料
* CSVファイル
* テーブル定義情報
* リンクテーブル情報
* クエリ情報
* VBA情報
* `.env`

確認後、移行対象となるAccessテーブルを整理してください。

---

### 2. Accessテーブル構造の確認

Access側の各テーブルについて、以下を確認してください。

* テーブル名
* カラム名
* Access上のデータ型
* 主キー
* インデックス
* NULL許可
* 既定値
* リンクテーブルか実テーブルか
* レコード件数

可能な範囲で正確に取得してください。

---

### 3. PostgreSQL用スキーマ作成

Accessの構造を忠実に再現するPostgreSQLスキーマを作成してください。

ただし、PostgreSQLでは日本語テーブル名・日本語カラム名を避け、英語名またはローマ字名に変換してください。

命名ルールは以下を基本としてください。

* テーブル名：スネークケース
* カラム名：スネークケース
* 英語化できるものは英語化
* 業務用語として英語化が難しいものはローマ字でも可
* 意味が不明なものは無理に意訳せず、元の日本語名が追跡できるようにする

例：

```text
外観検査記録 → appearance_inspection_records
品番 → product_code
品名 → product_name
指示日 → instruction_date
生産ロットID → production_lot_id
数量 → quantity
```

---

### 4. データ移行処理の作成・実行

AccessからPostgreSQLへデータを移行する処理を作成してください。

想定環境はWindowsです。

優先ライブラリ例：

* pyodbc
* sqlalchemy
* psycopg2
* pandas
* python-dotenv

実装時は以下を守ってください。

* 文字化け対策を行う
* 日付型の変換に注意する
* 空文字・NULLの扱いに注意する
* Yes/No型はbooleanへ変換する
* Accessの通貨型・数値型はPostgreSQLで適切な型にする
* 移行前後のレコード件数を比較する
* エラーが出た行はログに残す
* 再実行しやすい構成にする

---

### 5. 移行後チェック

移行後に以下を確認してください。

* Access側のテーブルごとの件数
* PostgreSQL側のテーブルごとの件数
* 件数差異の有無
* 移行失敗データの有無
* 型変換で注意が必要な項目
* NULLや空文字の扱いで注意が必要な項目

チェック結果をログまたはMarkdownに残してください。

---

### 6. 重要：移行対応表Markdownを必ず生成する

移行後に、以下のMarkdownファイルを必ず生成してください。

```text
対象フォルダ/migration_mapping.md
```

このファイルは、後で作成済みPythonアプリのフロント側をPostgreSQLへ接続するためにAIへ見せる重要資料です。

そのため、アプリ開発AIがこのファイルを読むだけで、

* どのPostgreSQL DBへ接続するのか
* AccessのどのテーブルがPostgreSQLのどのテーブルになったのか
* AccessのどのカラムがPostgreSQLのどのカラムになったのか
* 型がどう変換されたのか
* どのテーブルを参照すればよいのか

が分かるように、詳しく記載してください。

---

## migration_mapping.md に記載する内容

以下の構成で作成してください。

````markdown
# Access → PostgreSQL 移行対応表

## 1. 移行概要

- 対象Access DB：
- 移行先PostgreSQL DB：
- 接続情報：
  - `.env` の DATABASE_URL を参照
- 移行日：
- 作成者：
- 備考：

## 2. 移行対象テーブル一覧

| No | Accessテーブル名 | PostgreSQLテーブル名 | 種別 | Access件数 | PostgreSQL件数 | 備考 |
|---:|---|---|---|---:|---:|---|

## 3. テーブル別カラム対応表

### Accessテーブル名：〇〇
### PostgreSQLテーブル名：xxx_xxx

| No | Accessカラム名 | Access型 | PostgreSQLカラム名 | PostgreSQL型 | NULL許可 | 備考 |
|---:|---|---|---|---|---|---|

## 4. 主キー・インデックス情報

| Accessテーブル名 | PostgreSQLテーブル名 | 主キー | インデックス | 備考 |
|---|---|---|---|---|

## 5. 型変換ルール

| Access型 | PostgreSQL型 | 備考 |
|---|---|---|

例：

| Access型 | PostgreSQL型 | 備考 |
|---|---|---|
| Short Text | varchar / text | 文字列 |
| Long Text | text | 長文 |
| Number | integer / numeric | 内容に応じて判断 |
| Date/Time | timestamp / date | 日付のみの場合はdate |
| Yes/No | boolean | True/False |
| Currency | numeric | 金額 |
| AutoNumber | serial / integer | 主キー候補 |

## 6. アプリ接続時の参照情報

フロント側・既存Pythonアプリから参照する場合に必要な情報を記載してください。

### 接続先

```text
.env の DATABASE_URL を使用
````

### 主に参照するテーブル

| 用途 | PostgreSQLテーブル名 | 主なキー | 備考 |
| -- | --------------- | ---- | -- |

### 主要カラム

| 用途 | PostgreSQLテーブル名 | PostgreSQLカラム名 | 元Accessカラム名 | 備考 |
| -- | --------------- | -------------- | ----------- | -- |

## 7. 注意事項・要確認事項

* 移行時に判断に迷った項目
* 型変換で注意が必要な項目
* Access側に存在したがPostgreSQLで再現が難しい項目
* リンクテーブルに関する注意
* 後続アプリ開発時に注意すべき点


## 完了条件

以下を満たしたら完了としてください。

* Accessの対象テーブルがPostgreSQLへ移行されている
* 移行前後の件数確認ができている
* 差異やエラーが記録されている
* `migration_mapping.md` が生成されている
* `migration_mapping.md` を見れば、後続のアプリ開発AIがDB接続・テーブル参照・カラム参照を理解できる状態になっている

## 進め方

まず、対象フォルダの中身を確認してください。
その後、移行方針・対象テーブル・作成予定ファイルを簡単に整理してから実装してください。

勝手な統合・削除・省略はしないでください。
Access DBの内容をPostgreSQLへ忠実に再現することを最優先にしてください。
